# ID Aura — 设计师灵感图版 + 行业规范知识库

> **一款面向室内设计师的桌面灵感拼贴工具，内嵌 44 品类行业规范知识库，对标 PureRef 的核心交互体验。**


**ID Aura**（曾用名：设计规范灵感图版 / Moodboard App）是我为一个做餐饮工装设计的室内设计师朋友从零开发的桌面应用。它不是 AI 生成参考图——而是帮你 **组织参考图** 和 **查阅规范数据**，两者在同一块画布上完成。

> 如果你也是设计师，受够了在浏览器十几个标签页和微信文件之间来回穿梭整理灵感，这个工具就是为你做的。

---

## ✨ 功能亮点

### 🎨 自由灵感拼贴（对标 PureRef）
- **无限画布** — 缩放 10%–3200%，平移、缩放如行云流水
- **4 种导入方式** — 文件拖拽、文件选择器、剪贴板 Ctrl+V、URL 抓取
- **全自由变换** — 8 手柄缩放、旋转、移动，Shift 约束等比
- **框选多选** — 矩形框选 + Ctrl 单击追加/取消
- **6 方向对齐 + 等距分布** — 一键整理参考板布局
- **编组** — Ctrl+G 编组、双击进入组内编辑、组锁定
- **5 种排列模式** — 最优排列/按名称/按路径/按顺序/随机
- **撤销/重做** — 50 层快照栈，Ctrl+Z 随心回溯
- **自由绘制** — 画笔涂鸦 + 多色选择
- **文字标签** — 添加备注和文字标签

### 📐 行业规范知识库（核心差异化）
- **44 品类覆盖**：建筑设计 7 + 室内设计 30（餐饮工装 13 / 酒店民宿 7 / 家装 10）+ 景观设计 7
- **每条规范含**：名称、标准值、补充说明、优先级（高/中/低）
- **一键添加** — 规范条目作为可编辑卡片添加到画布
- **自定义规范** — 用户可以新增/编辑/删除自己的规范条目
- **双栏折叠菜单** — 品类按一级菜单折叠，快速定位
- **搜索** — 实时模糊搜索所有规范内容

### 🤖 AI 辅助（可选，BYO API Key）
- **自带 API Key 即可使用** — 支持 OpenAI / DeepSeek / Kimi / Qwen / GLM / Ollama / 自定义端点
- **流式对话** — 在侧面板直接和 AI 对话
- **规范上下文注入** — AI 自动获取当前品类的规范数据作为上下文
- **预设模板** — 检查清单/尺寸查询/布局建议/冲突排查/规范解释
- **三态防护** — 未配置/未测试/正常，安全可控

### 💾 本地优先，零服务器依赖
- **完全离线运行** — 所有数据存本地，不上传任何内容
- **.moodboard 文件** — 单文件保存/加载，图片内嵌 base64，可分发可备份
- **双击文件直接打开** — Windows 文件关联，和 .docx 一样方便
- **自动暂存** — localStorage 30 天自动保存，意外关闭不丢数据
- **导出 PNG** — 单张/批量导出，可用于方案汇报

## 🖥️ 技术栈

| 层 | 技术 |
|---|------|
| 前端框架 | React 18 + Vite 5 |
| 状态管理 | Zustand 4（7 slices） |
| 桌面壳 | Electron 28 |
| 图标 | lucide-react |
| 语言 | JavaScript（纯前端，无后端） |
| 构建 | electron-builder → NSIS Setup |

### 架构风格

**状态上移 + 命令下移 + 事件上报**：
- 所有状态集中在 Zustand Store（card / selection / canvas / project / history / settings / chat）
- UI 组件只负责渲染和事件上报，不维护业务状态
- Store action 内完成批量计算和更新

## 📦 下载与安装

### Windows 用户
从 [Releases](https://github.com/chengcheng067/Id-aura/releases) 页面下载最新的 `ID-Aura-*-Setup.exe`，双击安装即可。

> **系统要求**：Windows 10 或更高版本，64 位系统

### 从源码运行
```bash
# 克隆仓库
git clone https://github.com/your-username/id-aura.git
cd id-aura

# 安装依赖
npm install

# 开发模式（Vite 热更新 + Electron）
npm run electron:dev

# 生产构建 + 打包
npm run electron:build
```

## 🤝 如何参与共创

ID Aura 从一开始就是为真实用户解决真实问题而生的。它不是一个商业产品，而是一个**设计师和开发者共同打磨的工具**。无论你是设计师、开发者还是行业专家，都可以贡献你的力量。

### 1️⃣ 贡献规范数据（设计师 / 行业专家）

规范知识库是 ID Aura 的灵魂。目前我们覆盖了 30 个品类，但还远远不够。你可以：
- **补充现有品类的规范条目** — 如果你熟悉某个品类，提交 PR 补充数据
- **新增品类** — 我们还没有覆盖的餐饮/酒店/家装细分品类
- **修正错误** — 发现数据偏差或过时了？提 Issue 或直接改

数据在 `src/data/specs.js`，结构非常直观：
```js
const specs = {
  "category_key": {
    name: "品类名称",
    category: "餐饮工装", // 一级分类
    description: "品类简短描述",
    sections: [
      {
        title: "动线与通道", // 分类标题
        items: [
          { label: "主通道宽度", value: "≥ 1200mm", note: "消防疏散要求", priority: "high" },
        ]
      }
    ]
  }
}
```

### 2️⃣ 贡献代码（开发者）

我们欢迎任何形式的代码贡献：
- **Bug 修复** — 提 Issue 或直接 PR
- **功能增强** — 建议新功能前先提 Issue 讨论
- **性能优化** — 特别是 50+ 图片大画布场景
- **测试补充** — 用 vitest 写测试，简单友好

### 3️⃣ 贡献 AI 模型配置（AI 爱好者）

ID Aura 的 AI 功能是 BYO API Key 模式。如果你：
- **发现某个模型的兼容性问题** — 提 Issue
- **有推荐的模型/参数配置** — 提交默认参数 PR
- **想支持更多 API 格式** — 欢迎改进 `src/utils/aiClient.js`

### 4️⃣ 贡献设计 / 体验（设计师）

- **UI 文案** — 任何你觉得表达不够清晰的地方
- **交互优化** — 用起来不顺手的操作路径
- **图标/视觉** — 我们的图标可以更好看
- **翻译** — 多语言支持还在路上

### 🤔 不知道从哪开始？

可以查看 [Issues](../../issues) 标签为 `good first issue` 或 `help wanted` 的问题。如果不确定怎么做，先提 Issue 讨论就好。

## 💬 写在最后

这个项目始于一个简单的想法：**设计师不应该在找规范上浪费时间。**

我的设计师朋友丞丞说，每次做餐饮空间方案，光查防火规范、通道宽度、桌位配比就要翻半天资料。于是我们决定做一个工具——把"找图"和"查规范"放在一个地方，让灵感拼贴和规范检查在同一次操作中完成。

如果你有类似的需求，或者对这个项目有任何想法，欢迎提 Issue 或 PR。我们一起让它变得更好。

> **ID Aura — 集灵感，有规范。**
