# ID Aura v2.7.8 — 关闭按钮 + 进程名修复

## 🔧 修复 1：关闭按钮无响应

### 根因分析
关闭流程中有两个安全隐患：

1. **缺少错误兜底**：`close` 事件的 handler 调用 `executeJavaScript('window.__showCloseDialog__?.()')` 后没有 `.catch()`。如果 renderer 因任何原因（React 未挂载、组件异常等）未响应，**`e.preventDefault()` 已经阻止了窗口关闭，但没有对话框弹出 → 窗口完全卡死，点任何地方都无法关闭**

2. **async 竞态**：`close-dialog-response` 的 IPC handler 是 `async` 函数，存在 race condition 风险，且冗余调用了 `executeJavaScript('window.__saveProject__')`（renderer 已在保存后发 response）

### 修复方案
| 改动 | 文件 | 说明 |
|------|------|------|
| 新增 `closeDialogPending` 标志 | `electron/main.js` | 防止用户快速多次点击 X 时重入 |
| `executeJavaScript` 加 `.catch()` 兜底 | `electron/main.js` | renderer 无响应时强制 `destroy()` |
| `close-dialog-response` 改为同步 | `electron/main.js` | 移除 async，移除冗余保存，直接 `close()` |
| 点击"取消"发送 `'cancel'` IPC | `src/App.jsx` | 重置 pending 标志，允许再次点击关闭 |

### 关闭流程（修复后）
```
点击 X → close 事件 → closeDialogPending=true → e.preventDefault()
     → executeJavaScript('__showCloseDialog__') → 显示 CloseDialog
     → 用户点"不保存" → IPC 'discard' → forceQuitting=true → close() ✅
     → 用户点"保存并关闭" → renderer 先保存 → IPC 'save' → forceQuitting=true → close() ✅
     → 用户点"取消" → IPC 'cancel' → closeDialogPending=false → 窗口保持打开
     → executeJavaScript 失败 → catch → forceQuitting=true → destroy() ✅
```

## 🔧 修复 2：进程名不显示 ID Aura

### 根因
`package.json` 的 `name: "moodboard-app"` 在旧版本构建中没有 `productName` 配置时被 electron-builder 用作可执行文件名后备。任务管理器显示 `moodboard-app.exe` 而非 `ID Aura.exe`。

### 修复
`package.json` → `build.win` 新增 `artifactName: "ID-Aura-${version}-Setup.${ext}"`，确保产出命名统一。

验证：构建后 `release/win-unpacked/ID Aura.exe` ✅
