## 一、产品目标

1. **UI 风格从液态玻璃升级为 Linear/Notion 柔和卡片风格**，通过圆角容器、呼吸间距、去装饰化边框，提升设计师在长时间排版时的视觉舒适度，减少界面疲劳。
2. **规范库从单一室内设计扩展为建筑 + 室内 + 景观三大一级分类**，支撑工装设计师从更宏观的建筑视角到微观的景观视角进行全链路设计规范查询，增强产品专业壁垒。

## 三、功能需求池

### 3.1 P0 — 必须实现（v2.8.0 核心）

#### P0-1：UI 风格迭代 — Linear/Notion 柔和卡片风格

| 子功能 | 验收标准 |
|--------|----------|
| 圆角卡片容器 | 所有面板（工具栏、侧边栏、画布工具、状态栏）使用 `border-radius: 12px-16px` 的卡片容器包裹，替代当前的全宽/全高光边框风格。 |
| 呼吸间距体系 | 卡片间间距 ≥ 16px；卡片内 padding ≥ 16px；面板边缘距画布 ≥ 12px。整体感觉"不拥挤"。 |
| 去装饰化 | 移除或大幅弱化"虹彩边框"、"Aura 光晕"、"液态玻璃高光"等过度装饰；保留 subtle 的渐变 accent（蓝紫）仅用于主按钮和选中状态。 |
| 暗色基调保持 | 画布背景保持 `#1a1a1a`；卡片背景使用 `#1e1e24` / `#23232a` 的柔和暗色；无纯黑面板。 |
| 阴影层级 | 卡片使用柔和弥散阴影（`0 4px 24px rgba(0,0,0,0.2)`）替代高光边框；hover 时阴影加深 + 轻微抬升（`translateY(-2px)`）。 |
| 选中状态 | 卡片选中使用 2px 柔和蓝紫渐变边框（`#6b8cff → #a855f7`），无刺眼光晕。 |

#### P0-2：规范库重构 — 三级分类体系

| 子功能 | 验收标准 |
|--------|----------|
| 一级分类入口 | 侧边栏规范库顶部显示 3 个一级 tab：「建筑设计规范」「室内设计规范」「景观设计规范」。 |
| 室内设计规范保留 | 将现有 30 个品类（餐饮 13 + 酒店民宿 7 + 家装 10）完整归入「室内设计规范」下，数据零丢失。 |
| 建筑设计规范新增 | 新增 ≥6 个细分品类（办公建筑、商业建筑、教育建筑、医疗建筑、住宅建筑、工业建筑、交通建筑等），每个品类含完整的 sections/items。 |
| 景观设计规范新增 | 新增 ≥6 个细分品类（公园景观、商业景观、居住区景观、屋顶花园、水景设计、铺装设计、植物配置等），每个品类含完整的 sections/items。 |
| 品类切换 | 点击一级分类后，下方列出该分类下的所有细分品类（卡片或列表形式）；选中品类后展开 sections/items。 |
| 搜索跨库 | 搜索功能支持同时检索建筑、室内、景观三大分类下的所有规范条目。 |
| 数据结构兼容 | 现有已保存的 .moodboard 文件中的 spec 卡片，在升级后仍能正常读取和显示。 |

#### P0-3：AI 面板独立化

| 子功能 | 验收标准 |
|--------|----------|

## 四、UI 设计稿描述（文字版）

### 4.1 主画布区域

- **整体变化**：画布四周边缘与工具栏/侧边栏之间增加 `12-16px` 的呼吸间隙，工具栏和侧边栏不再贴边，而是像浮动卡片一样悬浮在画布之上。
- **工具栏（顶部）**：从全宽条状改为居中的圆角药丸卡片（`border-radius: 16px`，`padding: 8px 20px`），背景 `#23232a`，柔和阴影。内部按钮间距增大，分组间用竖线分隔而非色块。
- **缩放控件**：从右下角的紧凑按钮组改为圆角小卡片，按钮 hover 有 subtle 背景色变化。
- **状态栏（底部）**：改为左下角和右下角的两个小圆角卡片（如「缩放 85%」「图片 23 张」），不再是一条全宽横条。
- **图片卡片**：选中时的蓝色高光边框改为 2px 柔和渐变边框（`#6b8cff → #a855f7`），去掉白色方形手柄的过度突出，手柄改为与卡片融合度更高的圆角小点。
- **画布网格**：点状网格颜色从 `#2a2a2a` 微调为 `#252528`，更 subtle，不喧宾夺主。

### 4.2 侧边栏 — 规范库面板

- **整体容器**：从全高光边框的液态玻璃面板改为独立的圆角卡片（`border-radius: 16px`，`width: 300px`），与画布边缘有 12px 间距，背景 `#1e1e24`，阴影 `0 8px 32px rgba(0,0,0,0.25)`。
- **一级分类 Tab**：顶部用 3 个圆角胶囊按钮（pill tabs）代替目前的分组标题，当前选中项有 subtle 蓝紫底色，切换时有过渡动画。
- **品类列表**：当前分类下的细分品类用垂直列表展示，每项是一个小卡片（`border-radius: 10px`，`padding: 10px 14px`），hover 时有轻微抬升和背景色变化。
- **规范条目展开**：点击品类后展开 sections，每个 section 是一个可折叠的柔和卡片（带 chevron 图标），展开/收起有过渡动画。items 列表内每项右侧有「+」按钮，hover 显示。
- **搜索框**：顶部搜索框改为圆角输入框（`border-radius: 10px`），背景 `#23232a`，无边框，focus 时有 subtle 蓝紫 ring。

### 4.3 AI 独立面板

- **入口**：工具栏右侧新增「Sparkles」图标按钮；画布右下角增加一个 48px 的浮动圆形按钮（`border-radius: 50%`，蓝紫渐变背景，白色图标），hover 时轻微放大。
- **面板外观**：从屏幕右侧滑入，宽度 400px，圆角卡片（左上角/左下角 `16px`），背景 `#1e1e24`，顶部有拖拽条（可调整宽度）。
- **面板内部**：
  - 顶部标题栏：「AI 规范助手」+ 模式切换（快速问答 / 深度分析）+ 关闭按钮。
  - 对话区域：消息气泡采用柔和卡片样式，AI 消息左侧有 subtle 蓝紫竖线装饰；用户消息右侧对齐，背景 `#2a2a35`。
  - 输入区：底部固定输入框，圆角，带模板选择下拉和发送按钮。
  - 底部状态：显示当前使用的模型名称和基于的品类规范（如「基于：火锅店规范」）。
- **与规范库的关系**：AI 面板和 SidePanel 可同时存在，并排显示在画布右侧；两者之间有 12px 间距。

### 4.4 欢迎页（EmptyState）

- **变化**：去掉液态玻璃效果的中央大卡片，改为柔和的圆角内容区，背景 `#1e1e24`，`border-radius: 20px`。
- **文字排版**：标题使用更克制的字重（font-weight 500），描述文字颜色更柔和（`#a0a0b0`）。
- **快捷操作按钮**：从当前的霓虹渐变按钮改为柔和蓝紫渐变按钮（`border-radius: 12px`），hover 有轻微抬升。

## 六、数据结构变更说明

### 6.1 当前结构（v2.7.x）

```js
const specs = {
  "hotpot": {
    name: "火锅店",
    category: "餐饮工装",   // ← 仅用于二级分组
    icon: "🍲",
    description: "...",
    sections: [...]
  },
  ...
}
```

问题：
- `category` 字段仅为二级（餐饮工装/酒店民宿/家装），无法支撑三级分类。
- 新增的建筑、景观品类无法找到合适的 `category` 归属。

### 6.2 新结构（v2.8.0）

方案：**将现有 `category` 提升为二级 `subCategory`，新增一级 `category` 字段。**

```js
const specs = {
  "hotpot": {
    name: "火锅店",
    category: "室内设计规范",    // ← 一级：新增
    subCategory: "餐饮工装",     // ← 二级：原 category 改名
    icon: "🍲",
    description: "...",
    sections: [...]
  },
  "office_building": {
    name: "办公建筑",
    category: "建筑设计规范",    // ← 一级：新增
    subCategory: "办公建筑",     // ← 二级：自身即为细分
    icon: "🏢",
    description: "...",
    sections: [...]
  },
  "park_landscape": {
    name: "公园景观",
    category: "景观设计规范",    // ← 一级：新增
    subCategory: "公园景观",
    icon: "🌳",
    description: "...",
    sections: [...]
  },
  ...
}
```

### 6.3 兼容性策略

1. **存量 .moodboard 文件**：文件中的 `specData` 和 `sectionTitle` 不包含 `category` 字段，因此不受结构变更影响。加载后正常显示。
2. **自定义规范**：用户通过「+自定义规范」添加的条目，存储在 `customSpecs` 中，不依赖 `specs.js` 的一级分类，因此不受影响。
3. **`getCategoryList()` 函数**：需要在 `specs.js` 中升级，返回结构从 `[{ id, name, category, icon }]` 扩展为 `[{ id, name, category, subCategory, icon }]`。
4. **`getGroupedCategories()` 函数**：从按 `category` 分组改为按 `category`（一级）分组，UI 侧 SidePanel 先展示一级分类 Tab，再在一级内按 `subCategory` 展示二级分组。
5. **搜索函数 `searchSpecs()`**：保持原有逻辑不变，只需确保搜索范围覆盖新增的 14 个品类。

### 6.4 新增辅助函数建议

```js
// 获取所有一级分类（用于顶部 Tab）
const getPrimaryCategories = () => ["建筑设计规范", "室内设计规范", "景观设计规范"]

// 按一级分类获取品类列表
const getSpecsByPrimaryCategory = (primary) => 
  Object.entries(specs).filter(([_, v]) => v.category === primary)

// 按二级分类获取品类列表（保留兼容）
const getSpecsBySubCategory = (sub) => 
  Object.entries(specs).filter(([_, v]) => v.subCategory === sub)
```
